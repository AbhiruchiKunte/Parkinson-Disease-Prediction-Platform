# Security Policy

## Supported Versions
Currently, the latest version of **NeuroDetect** is actively maintained.  
Older versions may not receive security updates.

---

## Reporting a Vulnerability

If you discover a security vulnerability, please report it responsibly:

- Email: your-email@example.com  
- Or open a private issue (if enabled)

Please include:
- Description of the issue  
- Steps to reproduce  
- Possible impact  

⚠️ Do not publicly disclose the vulnerability before it is resolved.

---

## Response Policy

- We aim to respond within **48–72 hours**  
- Valid vulnerabilities will be addressed as soon as possible  
- You will be notified about the progress and resolution  

---

## Security Considerations

NeuroDetect is a healthcare-focused AI system. We prioritize:

- Secure handling of user input data (voice and clinical data)  
- Avoidance of storing sensitive personal information unless necessary  
- Use of trusted libraries and frameworks  
- Protection against common vulnerabilities (e.g., injection attacks, insecure file uploads)  

---

## Best Practices for Users

To use this project securely:

- Run the application in a secure environment  
- Do not upload sensitive real patient data unless properly authorized  
- Keep dependencies updated  
- Use secure API keys and environment variables  

---

## Disclaimer

This project is intended for **educational and research purposes only** and should not be used as a replacement for professional medical diagnosis.
